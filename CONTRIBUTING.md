# 🤝 기여 가이드 (Contributing Guidelines)

본 프로젝트는 4인 개발 체제(TA 1명, UE 프로그래머 2명, AI 엔지니어 1명)로 단기간(8월 26일 최종 발표) 내에 완성해야 하는 오픈월드 아포칼립스 생존 게임입니다. 파일 충돌을 최소화하고 효율성을 극대화하기 위해 아래 규칙을 엄격히 준수합니다.

## 👥 직무별 작업 영역 및 규칙 (Role Guidelines)

### 🎨 Technical Artist (TA - 1명)
- **작업 폴더:** `Content/Art/`, `Content/TechArt/` (머티리얼, 이펙트, 셰이더, 라이팅, 애니메이션)
- **에셋 적용:** 원본 작업 파일(`.psd`, `.blend`, `.spp` 등)은 용량 문제로 Git에 직접 올리지 않고 공용 드라이브 등에 보관합니다. Git에는 언리얼 엔진용으로 임포트된 에셋(`.uasset`)만 커밋합니다.
- **협업:** 마스터 머티리얼 등 공용 리소스를 수정할 때는 다른 팀원의 맵이 깨지지 않도록 사전에 공유합니다.

### 💻 UE Programmers (2명)
- **작업 폴더:** `Source/`, `Content/Blueprints/`
- **구조 원칙 (Component-Based):** 
  - 코어 로직(상태 관리, 인벤토리 등)은 C++ 액터 컴포넌트로 작성하여 독립적으로 작동하게 합니다.
  - 기획 및 TA가 수치를 조절할 수 있도록 Data Asset을 적극 활용합니다.
  - 두 명의 프로그래머가 서로 다른 컴포넌트 시스템을 담당하여 소스 코드 충돌을 예방합니다.
- **협업:** 이전 프로젝트에서 검증된 시스템(FSM, 컴포넌트 구조)을 최대한 재활용하여 개발 속도를 높입니다.

### 🤖 AI Engineer (1명)
- **작업 폴더:** `AI/` (LLM 스크립트, 서버 등), `Source/Companion/` (언리얼 연동부)
- **구조 원칙:** AI 동료의 로직(LLM 판단)과 언리얼 엔진(행동 트리 및 이동) 간의 책임을 명확히 분리합니다.
- **보안:** LLM API Key는 절대 Git에 커밋하지 않고 로컬의 `.env` 파일로만 관리합니다.

## 🌿 브랜치 전략 (Branch Strategy)
- `main` : 마일스톤 평가를 위한 안정적인 배포 브랜치
- `develop` : 개발용 메인 브랜치 (기본 기준점)
- `feature/[Role]/[Task]` : 개별 기능 브랜치 (예: `feature/TA/snow-shader`, `feature/UE/inventory`, `feature/AI/llm-prompt`)

## 📝 커밋 메시지 규칙 (Commit Convention)
- `[TA] 눈 덮인 지형 랜드스케이프 머티리얼 추가`
- `[UE] 플레이어 생존 스탯(허기, 갈증) 컴포넌트 추가`
- `[AI] LLM 프롬프트 최적화 및 파싱 로직 수정`

## ⚠️ 병합(Merge) 및 에셋 충돌 방지
1. **블루프린트 & 레벨 에셋 독점:** 언리얼 바이너리 에셋(`.uasset`, `.umap`)은 Merge가 불가능합니다. 수정 전 반드시 팀 메신저에 "OOO 블루프린트 작업 시작합니다"라고 알리고 독점권(Lock)을 가집니다.
2. **독립된 테스트 맵 사용:** 각자 자신만의 샌드박스 레벨(`Content/Maps/Test/[Name]_TestMap.umap`)에서 기능을 개발 및 검증한 후, 최종적으로 메인 레벨에 통합합니다.
